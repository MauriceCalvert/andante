## Task: V5 — Wire viterbi into invention follower

Read these files first:
- `viterbi/adapter.py` (from V4)
- `builder/phrase_writer.py`
- `builder/imitation.py`
- `builder/phrase_types.py`

### Goal

In inventions, the voice NOT carrying the subject is the follower. Wire
the viterbi solver for this case. The subject-carrying voice is the leader
(its notes are already generated by the time the follower runs).

### Implementation

The `imitation_role` field in `PhrasePlan` determines which voice carries
the subject. The other voice is the follower.

**When the soprano carries the subject:**
- Leader = soprano (already generated)
- Follower = bass
- Follower range = bass range
- Corridors built with soprano as leader

**When the bass carries the subject:**
- Leader = bass (already generated)
- Follower = soprano
- Same as the galant case (V4)

The adapter from V4 is bidirectional — it converts whichever finished
voice into the leader surface. The solver doesn't care which voice is
which; it only knows leader and follower.

In `phrase_writer.py`, the dispatch that currently calls
`generate_soprano_phrase` vs `generate_bass_phrase` gains a third path:
for invention phrases where the follower uses viterbi.

### CostProfile selection

Use the `INVERTIBLE_PROFILE` (from V4's CostProfile addition) when the
invention requires invertible counterpoint — i.e. when both voices share
the same register and crossing is expected. Detect this from the phrase
plan: if both voices have overlapping ranges (the follower range overlaps
the leader range by more than an octave), pass `INVERTIBLE_PROFILE`.
Otherwise pass `DEFAULT_PROFILE`.

For a simple first pass: always use `INVERTIBLE_PROFILE` for invention
follower voices. Inventions are written to be invertible; galant pieces
are not. This can be refined later if needed.

### Constraints

- Do not modify imitation.py (subject/answer placement is untouched).
- Do not modify bass_writer.py's own generation logic.
- The solver for the bass-as-follower case uses the bass range, not the
  soprano range. Pass the correct range bounds.
- Voice crossing is permitted but costly (existing cost function handles
  this).

### Checkpoint

Generate an invention phrase where the bass carries the subject. Verify
the soprano (follower) produces a shaped counterpoint line against it.
Then generate a phrase where the soprano carries the subject and verify
the bass follower.
